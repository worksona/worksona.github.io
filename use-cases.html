Customer Support Chatbot

Create a customer support chatbot that can handle common inquiries, provide product information, and assist with basic troubleshooting.

Implementation

// Define the customer support agent
const supportAgent = {
  id: 'customer-support',
  name: 'Sarah',
  description: 'Friendly customer support specialist',
  traits: {
    personality: ['empathetic', 'patient', 'solution-oriented'],
    knowledge: ['product catalog', 'return policies', 'shipping information'],
    tone: 'warm and professional'
  },
  config: {
    provider: 'openai',
    model: 'gpt-3.5-turbo',
    temperature: 0.7,
    systemPrompt: 'You are Sarah, a friendly customer service rep...'
  }
};

// Load the agent
await worksona.loadAgent(supportAgent);

// Handle user messages
async function handleUserMessage(message) {
  try {
    const response = await worksona.chat('customer-support', message);
    displayResponse(response);
  } catch (error) {
    handleError(error);
  }
}

Best Practices

  • Define clear personality traits and knowledge domains for consistent responses
  • Include example conversations in the system prompt
  • Implement error handling and fallback responses
  • Use the conversation history for context-aware responses

# Customer Support Chatbot Implementation Prompt

## Overview
Create a customer support chatbot application using Worksona.js that provides automated customer service capabilities. The application should leverage AI agents to handle customer inquiries, provide product information, and assist with basic troubleshooting.

## Technical Requirements
- Framework: Worksona.js for AI agent management and interaction
- Agent Configuration: Customer support agent with empathetic personality and product knowledge
- UI: Clean, responsive chat interface
- Features: Real-time chat, conversation history, typing indicators

## Worksona.js Configuration
```yaml
# worksona-config.yaml
version: 1.0.0

core:
  debug: true
  controlPanel: true
  defaultProvider: "openai"
  maxHistoryLength: 10
  eventHandling: true
  errorReporting: true

providers:
  openai:
    models:
      - name: "gpt-3.5-turbo"
        maxTokens: 4096
        temperature: 0.7
    rateLimits:
      requestsPerMinute: 60
      tokensPerMinute: 90000

events:
  chat:
    - name: "chat-start"
      data:
        agentId: string
        message: string
    - name: "chat-complete"
      data:
        agentId: string
        response: string
    - name: "chat-error"
      data:
        agentId: string
        error: object
```

Implementation

const contentAgent = {
  id: 'content-writer',
  name: 'Alex',

    personality: ['creative', 'concise', 'engaging'],
    knowledge: ['SEO best practices', 'content marketing'],
    tone: 'professional and engaging'
  },
  config: {
    provider: 'anthropic',
    model: 'claude-2',
    temperature: 0.8,
    maxTokens: 1000
  }
};

// Load the agent
await worksona.loadAgent(contentAgent);

// Generate content
async function generateContent(prompt, type) {

}

  • Consider using different agents for different content types

Document Processing

Create an agent that can analyze, summarize, and extract information from documents.

Implementation

// Define the document processing agent
const docAgent = {
  id: 'document-processor',
  name: 'Doc',
  description: 'Document analysis specialist',
  traits: {
    personality: ['analytical', 'precise', 'thorough'],
    knowledge: ['document analysis', 'information extraction'],
    tone: 'formal and precise'
  },
  config: {
    provider: 'openai',
    model: 'gpt-4',
    temperature: 0.3,
    maxTokens: 2000
  }
};

// Load the agent
await worksona.loadAgent(docAgent);

// Process a document
async function processDocument(documentText, task) {
  const message = `Please ${task} the following document:\n\n${documentText}`;
  const result = await worksona.chat('document-processor', message);
  return result;
}

Best Practices

  • Break down complex documents into smaller chunks
  • Implement validation and verification steps

Multi-Agent Systems

Create a system of multiple agents that can work together to solve complex tasks.

Implementation

// Define multiple agents
const agents = [
  {
    id: 'researcher',
    name: 'Research',
    description: 'Information gathering specialist',
    config: {
      provider: 'openai',
      model: 'gpt-4'
    }
  },
  {
    id: 'analyst',
    name: 'Analysis',
    description: 'Data analysis specialist',
    config: {
      provider: 'anthropic',
      model: 'claude-2'
    }
  },
  {
    id: 'writer',
    name: 'Writer',
    description: 'Content creation specialist',
    config: {
      provider: 'openai',
      model: 'gpt-3.5-turbo'
    }
  }
];

// Load all agents
for (const agent of agents) {
  await worksona.loadAgent(agent);
}

// Coordinate multiple agents
async function coordinateAgents(task) {
  // Research phase
  const research = await worksona.chat('researcher', task);
  
  // Analysis phase
  const analysis = await worksona.chat('analyst', research);
  
  // Writing phase
  const finalContent = await worksona.chat('writer', analysis);
  
  return finalContent;
}

Best Practices

  • Define clear roles and responsibilities for each agent
  • Implement proper error handling and fallback mechanisms
  • Use different providers/models for different tasks
  • Consider implementing a coordinator agent

Image Generation and Analysis Demo

This demo shows how to generate an image from a prompt and then analyze it using the same agent.

const worksona = new Worksona({
  apiKeys: { openai: 'sk-...' }
});

await worksona.loadAgent({
  id: 'creative-agent',
  name: 'Creative Agent',
  config: { provider: 'openai', model: 'gpt-4o' }
});

// Generate an image
const generatedImageUrl = await worksona.generateImage('creative-agent', 'A futuristic cityscape at sunset');
console.log('Generated Image URL:', generatedImageUrl);

// Analyze the generated image
const analysis = await worksona.analyzeImage('creative-agent', generatedImageUrl);
console.log('Analysis:', analysis);

Chained Analysis and Generation Demo

This advanced demo chains image analysis and generation. It analyzes an input image, generates a new image based on the analysis, and then analyzes the generated image.

const worksona = new Worksona({
  apiKeys: { openai: 'sk-...' }
});

await worksona.loadAgent({
  id: 'creative-agent',
  name: 'Creative Agent',
  config: { provider: 'openai', model: 'gpt-4o' }
});

// Step 1: Analyze an input image
const inputImageUrl = 'https://example.com/input-image.png';
const initialAnalysis = await worksona.analyzeImage('creative-agent', inputImageUrl);
console.log('Initial Analysis:', initialAnalysis);

// Step 2: Generate a new image based on the analysis
const prompt = `Create an image inspired by this analysis: ${initialAnalysis}`;
const newImageUrl = await worksona.generateImage('creative-agent', prompt);
console.log('New Image URL:', newImageUrl);

// Step 3: Analyze the newly generated image
const newImageAnalysis = await worksona.analyzeImage('creative-agent', newImageUrl);
console.log('New Image Analysis:', newImageAnalysis);
